# Render Blueprint Deploy Notes (Prompt 18)

## Scope and assumptions

- This repository currently contains:
  - production backend (`backend/`)
  - web static shell (`web/shell`) used as first-stage frontend runtime
- A full Next.js app is not present in this repo yet, so Render frontend is configured as a `static` service.

## What was hardened for deploy

- Added Node-based migrations runner (no `psql` binary dependency on runtime image):
  - `backend/scripts/migrate.mjs`
- Added super-admin bootstrap script:
  - `backend/scripts/bootstrap-super-admin.mjs`
- Added startup/readiness hardening:
  - request logging middleware in `backend/src/app.ts`
  - `GET /api/v1/readyz` with db/integration status
  - startup integration state logs in `backend/src/main.ts`
- Added robust trust proxy parsing (`TRUST_PROXY=true` no longer crashes startup).
- Added static web build pipeline with backend URL injection:
  - `web/scripts/build-static.sh`
  - generated `web/dist/assets/config.js` from `PUBLIC_API_BASE`

## Blueprint

- File: `render.yaml`
- Provisions automatically:
  - `autoservice-postgres` database
  - `autoservice-backend` web service
  - `autoservice-web` static service

## Environment variables categories

### A) Autogenerated in blueprint

- `JWT_SECRET` (`generateValue`)
- `JWT_REFRESH_SECRET` (`generateValue`)
- `APP_SECRET` (`generateValue`)
- `SESSION_SALT` (`generateValue`)
- `SUPER_ADMIN_PASSWORD` (`generateValue`)

### B) From database/service links

- `DATABASE_URL` (`fromDatabase.connectionString`)
- `PUBLIC_API_BASE` (`fromService.host` for backend; build script adds `https://` and appends `/api/v1`)
- `CORS_ORIGIN` (`fromService.host` for web; backend normalizes it to a valid origin)

### C) Defaults / safe automatic values

- `NODE_ENV=production`
- `ACCESS_TOKEN_TTL=15m`
- `REFRESH_TOKEN_TTL_DAYS=30`
- `TRUST_PROXY=true`
- `CORS_EXTRA_ORIGINS=` при необходимости можно добавить дополнительные origin вручную
- `FILES_ENABLED=false`
- `NOMINATIM_*` and cache TTL defaults
- `SUPER_ADMIN_EMAIL=admin@autoservice.local`
- `SUPER_ADMIN_NAME=System Super Admin`

### D) Disabled features (degraded but working)

- `FCM_SERVER_KEY=""` → push delivery is disabled gracefully (no startup crash).
- `FILES_ENABLED=false` + empty S3 vars → file upload endpoints return controlled error.
- `PUSH_ENABLED=false` → push flow disabled at service layer.
- `SMTP_ENABLED=false` → email channel disabled for stage-1 deploy.

### E) External secrets to add later (optional)

- `FCM_SERVER_KEY` (enable push)
- `S3_*` + `FILES_ENABLED=true` (enable attachments storage)

## Deploy order / minimal manual steps

1. Import `render.yaml` in Render.
2. Deploy services.
3. Open backend `/api/v1/readyz` and confirm `status=ready`.
4. Login with bootstrap account:
   - email: `luberm6@gmail.com`
   - password: generated in Render env var `SUPER_ADMIN_PASSWORD`
5. Immediately change password after first login.
6. After first successful login, you can remove `SUPER_ADMIN_PASSWORD` from Render env (keep `SUPER_ADMIN_EMAIL`).

## Known limitations

- Web service is static shell for stage-1 runtime; migrate to Next.js service when Next app is added to repository.
- Mobile map module currently exposes branch geography as an operational panel, not a full native interactive map runtime.
