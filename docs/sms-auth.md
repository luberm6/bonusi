# SMS-авторизация в Centr Radius Service

В проекте реализована двухфакторная/беспарольная авторизация по SMS-коду через сервис [SMSC.ru](https://smsc.ru/), которая работает параллельно с классической авторизацией по Email и паролю.

---

## 1. Принцип работы SMS-авторизации

SMS-вход состоит из двух шагов:

1. **Запрос кода (`POST /api/v1/auth/otp/request`):**
   * Клиент указывает номер телефона.
   * Номер нормализуется (приводится к единому международному формату, например, `+79991234567`).
   * Генератор создает 6-значный одноразовый код (длина настраивается через `SMS_OTP_LENGTH`).
   * Хеш кода (SHA-256) записывается в таблицу `sms_otp_codes` со временем жизни (по умолчанию 5 минут / 300 секунд).
   * Все старые неиспользованные коды для этого телефона аннулируются (`consumed_at = now()`).
   * Накладывается ограничение (Rate limit): повторный запрос возможен не чаще, чем раз в 60 секунд.
   * Код отправляется пользователю через SMSC.ru или выводится в консоль (в режиме Mock/тестирования).

2. **Проверка кода (`POST /api/v1/auth/otp/verify`):**
   * Пользователь вводит полученный код.
   * Сервер проверяет актуальность кода (не истек ли срок действия, не был ли он уже использован, не превышен ли лимит попыток ввода — макс. 5 попыток).
   * Если код верный, сервер находит пользователя по столбцу `phone_number` или `phone` в таблице `public.users`.
   * При первом успешном входе проставляется дата верификации телефона `phone_verified_at = now()`, а также обновляется `last_sms_login_at`.
   * Пользователю выпускается стандартная сессия с JWT `accessToken` и `refreshToken`, аналогичная обычному входу.

## 2. API Эндпоинты

### Запрос SMS-кода
* **URL:** `POST /api/v1/auth/otp/request`
* **Тело запроса:**
  ```json
  {
    "phone": "+79991234567"
  }
  ```
* **Успешный ответ:**
  ```json
  {
    "success": true,
    "message": "Code sent"
  }
  ```

### Проверка SMS-кода и вход
* **URL:** `POST /api/v1/auth/otp/verify`
* **Тело запроса:**
  ```json
  {
    "phone": "+79991234567",
    "code": "123456",
    "device": {
      "deviceId": "optional-device-uuid",
      "platform": "ios",
      "deviceName": "iPhone 11",
      "appVersion": "1.0.0"
    }
  }
  ```
* **Успешный ответ:**
  ```json
  {
    "accessToken": "ey...",
    "refreshToken": "...",
    "user": {
      "id": "user-uuid",
      "email": "client@mail.ru",
      "role": "client",
      "isActive": true
    },
    "deviceId": "device-uuid"
  }
  ```

## 3. Переменные окружения (.env / Render)

Для работы SMS-авторизации необходимо настроить следующие переменные окружения:

```env
# Включение функционала на бэкенде
SMS_OTP_ENABLED=true

# Выбор провайдера отправки: 'smsc' для реальной отправки, 'mock' для логирования в консоль
SMS_PROVIDER=smsc

# Настройки одноразовых кодов
SMS_OTP_LENGTH=6
SMS_OTP_TTL_SECONDS=300
SMS_OTP_RESEND_SECONDS=60
SMS_OTP_MAX_ATTEMPTS=5

# Настройки интеграции с SMSC.ru
SMSC_LOGIN=ваш_логин_в_smsc
SMSC_PASSWORD=ваш_пароль_в_smsc (или MD5-хеш пароля)
SMSC_SENDER=имя_отправителя (должно быть зарегистрировано в SMSC.ru, по умолчанию пустое)
SMSC_ENDPOINT=https://smsc.ru/sys/send.php
SMSC_USE_POST=true
SMSC_RESPONSE_FORMAT=json
SMSC_DEBUG=false

# Тестовый режим (для Apple Review и локальной отладки)
TEST_SMS_ENABLED=true
TEST_SMS_PHONE=+79990000000
TEST_SMS_CODE=123456
```

## 4. Режим тестирования и Apple Review

Для успешного прохождения модерации в App Store/TestFlight (а также для локальной отладки без реальной отправки платных SMS) предусмотрен тестовый обходной путь:

1. Установите `TEST_SMS_ENABLED=true`.
2. Задайте `TEST_SMS_PHONE` (например, `+79990000000`) и `TEST_SMS_CODE` (например, `123456`).
3. При запросе кода на этот номер телефона бэкенд не отправляет реальный запрос на SMSC.ru, а сразу возвращает `{"success":true,"message":"Code sent"}`.
4. При проверке кода для этого номера сервер примет `123456` как верный код и авторизует пользователя.

*Для Apple Reviewer-ов также продолжает полностью функционировать вход по Email/паролю:*
* **Email:** `apple-review@centradius.service`
* **Пароль:** `AppleReview2026!`

---

## 5. Как переключить с тестового аккаунта SMSC.ru на аккаунт клиента

Когда клиент предоставит свои данные от аккаунта SMSC.ru, вам нужно будет только обновить переменные окружения на сервере (в панели управления Render):

1. Измените `SMSC_LOGIN` на логин клиента в SMSC.ru.
2. Измените `SMSC_PASSWORD` на пароль от аккаунта клиента (или MD5-хеш пароля, сгенерированный в личном кабинете SMSC.ru для безопасности).
3. При необходимости пропишите зарегистрированное клиентом буквенное имя отправителя в `SMSC_SENDER`.
4. Перезапустите веб-сервис на Render, чтобы новые настройки применились. Никаких изменений в коде делать не потребуется!
